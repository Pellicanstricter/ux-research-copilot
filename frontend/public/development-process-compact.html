A step-by-step development journey

1. Problem Definition

Identified the time-consuming aspect of translating user feedback into presentations. UX researchers spend hours manually analyzing transcripts and creating slides.

2. System Architecture & AI Setup

Used AI to start building the app. I used Claude Code within my terminal to create code locally‚Äîthis worked much better than copy-pasting code in Visual Studio Code and streamlined the development process.

3. Core Development

Built file upload system supporting PDF, DOCX, TXT, and CSV. Created FastAPI backend with async processing and React frontend with 16:9 presentation slides.

4. AI Integration

Integrated LangChain with OpenAI GPT-4 to create multi-agent system. AI automatically clusters themes (3-5 max), extracts supporting quotes, generates insights with priority ranking, and creates actionable recommendations.

5. Iterative Refinement

Refined AI prompts through testing to limit themes and improve accuracy. Added edit/delete functionality for all slides. Applied professional design patterns with color-coded sections and improved visual hierarchy.

6. User Experience Polish

Created sample report demo for first-time users. Added modal editing so users can modify AI-generated content. Implemented auto-generated research goals from themes.

7. Production Deployment

Deployed to Railway with automatic CI/CD from GitHub. Tested with real transcripts and validated <5 minute processing time. Set up Redis for session management and result persistence.

8. Critical Evaluation

Recognized key limitations: AI output requires human validation, no screenshot/visual asset integration, no direct PowerPoint export. Future improvements include PPTX export, image upload support, and confidence scoring for insights.
