üí° Key Technical Decisions

Why LangChain + OpenAI GPT-4?

Decision: Multi-agent LangChain architecture with GPT-4

Rationale: LangChain provides excellent orchestration for complex AI workflows. GPT-4 offers superior natural language understanding for theme clustering and insight extraction. The multi-agent approach allows specialized prompts for each task (themes, insights, recommendations).

Why FastAPI over Flask?

Decision: FastAPI with async background tasks

Rationale: FastAPI's native async support enables non-blocking file processing. Background tasks allow UI to remain responsive during AI analysis. Automatic API documentation and type validation reduce bugs.

Why Redis for Session Management?

Decision: Redis for session state and results caching

Rationale: Redis provides fast in-memory storage for session tracking during async processing. Allows users to check status and retrieve results after background tasks complete. Enables saved reports feature.

Why 16:9 Slide Format?

Decision: Presentation-ready 16:9 aspect ratio slides

Rationale: UX researchers need to present findings to stakeholders. 16:9 format is industry standard for presentations. Direct screenshot-to-PowerPoint workflow saves significant time.

üéØ AI Output Quality

Limitation: AI-generated insights are not always perfect

GPT-4 can occasionally misinterpret context, miss nuanced user sentiments, or generate themes that don't fully align with research intent. The quality heavily depends on input transcript clarity and structure.

Future Work: Implement confidence scoring for insights, add human review checkpoints, fine-tune prompts with domain-specific examples

‚úÖ Human Validation Required

Limitation: Researchers must validate all AI outputs

The tool accelerates the process but cannot replace human expertise. Researchers need to verify themes, validate quotes in context, and ensure recommendations align with business goals. AI is an assistant, not a replacement.

Future Work: Add validation checklists, implement peer review workflows, provide source traceability for every insight

üì∏ No Screenshot Integration

Limitation: Cannot incorporate presentation screenshots or visual assets

Current output is text-only slides. Researchers often need to include app screenshots, user journey diagrams, or visual mockups in presentations. These must be added manually after export.

Future Work: Allow image uploads during processing, integrate screenshot annotation tools, support visual assets in slide templates

üìä Limited Export Formats

Limitation: No direct PowerPoint/Google Slides export

Users currently screenshot slides or manually recreate them in presentation tools. While 16:9 format helps, there's no automated export to PPTX/Google Slides with proper formatting.

Future Work: Build PPTX export with python-pptx, create Google Slides API integration, add PDF export with embedded fonts

üîç Context Window Limitations

Limitation: Large transcript sets may exceed AI context limits

GPT-4 has token limits. Very long transcripts (20+ interviews) may need chunking, which can cause the AI to miss cross-interview patterns or themes that span multiple documents.

Future Work: Implement hierarchical summarization, use retrieval-augmented generation (RAG), explore Claude's 200k context window

üí∞ Cost Considerations

Limitation: OpenAI API costs scale with usage

Each analysis uses GPT-4 tokens for multiple agents (themes, insights, recommendations). High-volume usage can become expensive. Currently no cost optimization or caching beyond Redis session storage.

Future Work: Implement intelligent caching, use GPT-3.5 for preliminary analysis, add usage monitoring and cost alerts
